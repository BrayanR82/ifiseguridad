Despliegue de la web con Docker

Guía sencilla para subir una web usando Docker, de forma rápida, ordenada y fácil de repetir.

Pensada para evitar problemas entre desarrollo y producción, y poder volver atrás fácilmente si algo falla.

Antes de empezar

  • Docker instalado y funcionando.
  • Docker Compose instalado.
  • Código de la web listo (proyecto).
  • Archivo Dockerfile preparado.
  • Archivo docker-compose.yml listo.
  • Puertos definidos (ej: 80, 3000, etc.).

Paso a paso

  1. Abre la terminal en la carpeta del proyecto.
  2. Construye la imagen de Docker:
    docker build -t mi-web .
  3. Levanta los contenedores con Docker Compose:
    docker-compose up -d
  4. Verifica que los contenedores están corriendo:
    docker ps
  5. Accede a la web desde el navegador usando la IP o localhost.
  6. Si usas base de datos o archivos, asegúrate de que los datos se guardan (volúmenes).
  7. Revisa logs si algo falla:
    docker-compose logs
  8. Si necesitas parar:
    docker-compose down
  9. Si haces cambios, reconstruye y vuelve a levantar:
    docker-compose up --build -d
  10. Comprueba que todo funciona antes de darlo por terminado.

Buenas prácticas

  • Guardar datos importantes en volúmenes (no dentro del contenedor).
  • Revisar logs cuando algo no funcione.
  • No abrir más puertos de los necesarios.
  • Probar la web después de cada despliegue.
  • Tener copia del proyecto antes de hacer cambios grandes.
  • Parar contenedores que no se estén usando.
  • Mantener Docker actualizado.
