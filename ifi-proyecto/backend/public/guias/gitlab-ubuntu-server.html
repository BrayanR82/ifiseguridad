Subida de repositorio a GitLab desde Ubuntu Server

Guía sencilla para subir un proyecto a GitLab desde Ubuntu Server usando Git, SSH y tokens de acceso.

Pensada para trabajar desde servidor, subir código de forma segura y mantener el repositorio actualizado sin complicaciones.

Antes de empezar

  • Servidor Ubuntu con acceso por SSH.
  • Git instalado (sudo apt install git).
  • Cuenta en GitLab y repositorio creado.
  • Token de acceso o clave SSH configurada.
  • Proyecto listo en el servidor.

Paso a paso

  1. Accede al servidor por SSH.
  2. Entra en la carpeta del proyecto.
  3. Inicializa Git:
    git init
  4. Añade los archivos:
    git add .
  5. Haz el primer commit:
    git commit -m "primer commit"
  6. Copia los comandos que te da GitLab al crear el repositorio.
  7. Añade el repositorio remoto:
    git remote add origin URL_DEL_REPO
  8. Sube el proyecto:
    git push -u origin main
  9. Si pide acceso:
    • Usa token (usuario + token como contraseña)
    • O usa SSH si ya está configurado
  10. Verifica en GitLab que los archivos se han subido correctamente.

Buenas prácticas

  • Usar SSH en lugar de usuario/contraseña para más seguridad.
  • Guardar el token en un lugar seguro (no compartirlo).
  • Subir cambios con frecuencia para no perder trabajo.
  • Verificar siempre en GitLab que todo esté correcto.
  • Probar conexión antes de trabajar para evitar errores.
