Asignación de direcciones IP a dispositivos

Estándar de asignación IP para reducir conflictos, mejorar mantenimiento y facilitar la escalabilidad de sistemas de seguridad.

Un direccionamiento bien planificado acelera soporte, reduce tiempos de diagnóstico y permite integrar nuevos dispositivos sin interrupciones.

Asignación de direcciones IP con DHCPsrv

Configuración de un servidor DHCP local para asignar direcciones IP automáticamente a dispositivos de red.

El uso de DHCPsrv permite centralizar la asignación de IP, reducir errores manuales y agilizar la incorporación de nuevos equipos en la red.

Antes de empezar

  • Descarga DHCPsrv 2.5.2 desde su web oficial.
  • Ejecuta el programa con permisos de administrador en un equipo conectado a la red.
  • Asegúrate de tener acceso al router (gateway) y conocer el rango de IP de tu red.

Paso a paso

  1. Descomprime el archivo descargado y ejecuta dhcpsrv.exe.
  2. Abre el archivo de configuración dhcpsrv.ini con un editor de texto.
  3. Define la máscara de red correspondiente.
  4. Configura el rango de direcciones IP que se asignarán automáticamente.
  5. Establece la puerta de enlace (gateway) de la red.
  6. Configura los servidores DNS.
  7. Guarda los cambios realizados en el archivo.
  8. Instala el servicio DHCP (opcional).
  9. Inicia el servidor DHCP.
  10. Configura los dispositivos cliente en modo automático (DHCP).

Comandos útiles

# Ejecutar servidor
dhcpsrv.exe

# Instalar como servicio
dhcpsrv.exe -install

# Iniciar servicio
net start dhcpsrv

Ejemplo de configuración

Archivo base recomendado para dhcpsrv.ini. Puedes copiarlo y adaptarlo al rango, gateway y DNS de tu red.

# Archivo: dhcpsrv.ini

[GENERAL]
SUBNETMASK=255.255.255.0

[RANGE_SET]
DHCPRANGE=192.168.1.100-192.168.1.200

# Puerta de enlace
ROUTER_0=192.168.1.1

# Servidores DNS
DNS_0=8.8.8.8
DNS_1=8.8.4.4

Buenas prácticas

  • No tener más de un servidor DHCP activo en la misma red.
  • Definir rangos de IP fuera del DHCP para dispositivos críticos (routers, NVR, servidores).
  • Documentar siempre el rango utilizado y la configuración aplicada.
  • Supervisar conflictos de IP y revisar logs del servidor si es necesario.
